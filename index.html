Mooncore

Lightweight, action-based API framework for Elixir — a Phoenix alternative that puts actions at the center of everything.

Introduction

Mooncore is a lightweight, action-based API framework for Elixir — a Phoenix alternative that puts actions at the center of everything.

Why Mooncore?

Phoenix is powerful but often brings a heavy ecosystem. Mooncore takes a different path: it centers your app around named actions instead of controllers and heavy frameworks.

Core Ideas

  • Actions, not controllers: Named actions (e.g., "task.create", "user.login") map to handler functions — no controllers or per-endpoint routes required.
  • Transport-agnostic: Actions are callable from HTTP, WebSocket, or direct Elixir calls without code changes.
  • You own the router: Mooncore provides building blocks; you decide your Plug.Router and response shapes.
  • No database opinions: Pick your database client and wire it through middleware; Mooncore makes no assumptions.
  • Minimal magic: No code generation or hidden macros — explicit, visible control flow.

What It Includes

Component Purpose
Mooncore.Action Action dispatcher with role checking and middleware pipeline
Mooncore.Auth JWT RS256 authentication (token creation, verification, plug)
Mooncore.Endpoint.Http HTTP adapter — turns a Plug.Conn into an action call
Mooncore.Endpoint.Socket WebSocket adapter — pub/sub broadcasting and message routing
Mooncore.Middleware Before/after hooks for request/response transformation
Mooncore.App Multi-app registry — support multiple apps in one deployment
Mooncore.Dev Development dashboard and MCP server for AI observability

What It Doesn't Include

  • No HTML templating engine
  • No database layer or ORM
  • No asset pipeline
  • No code generators
  • No LiveView equivalent

You bring tools for those needs; Mooncore focuses on action dispatch, auth, real-time comms, and middleware.

Who Is It For?

  • Developers building API-first applications and real-time backends
  • Teams that want WebSocket support as a first-class feature
  • Projects favoring explicit code over convention-heavy frameworks
  • Multi-tenant or multi-app deployments

Learn more in the repository README or the docs in the guides/ folder.
